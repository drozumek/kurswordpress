  • Plik wp-config.php, który łączy wordpress z bazą danych

Dzięki temu elementy, których treść się nie zmienia, takie jak stopka czy nagłówek, są dynamicznie dodawane do każdej podstrony naszego motywu - nie musimy ich kopiować za każdym razem na podstrony. Wystarczy, że zmodyfikujemy jeden plik footer.php a stopki na wszystkich podstronach naszej witryny zostaną zmienione.

Przede wszystkim nie zniechęcaj się po zobaczeniu jak wiele jest tutaj plików. Wordpress to przemyślany i prosty do nauczenia system. Jeśli będziesz przestrzegał zaleceń wordpressa dotyczących budowy własnego motywu, to wszystko będzie działać sprawnie - silnik wordpressa rozpozna odpowiednie pliki i zajmie się logiką ich przetwarzania w odpowiednim momencie i wyświetlania odpowiedniej zawartości Twojej strony.

Pierwszym plikiem, który możemy/powinniśmy stworzyć w katalogu głównym motywu jest plik "style.css". Plik ten zawierać będzie m.in. niezbędne informacje, które pozwolą na poprawne wyświetlenie się motywu w panelu administracyjnym Wordpressa w sekcji "Motywy".

AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA Your CSS stylesheets are used to customize the presentation of your theme. A stylesheet is also the file where information about your theme is stored. For this reason, the style.css file is required in every theme.

Tworzymy główny plik ze stylami

  1. W katalogu głownym motywu tworzymy plik style.css
  2. W utworzonym pliku w wpisujemy jako komentarz /* */: 
    /*
Theme Name: Super Motyw // Nazwa naszego motywu
Theme URI: http://supermotyw.com
Author: Damian Rozumek
Author URI: www.strona-autora.pl
Description: Świetny motyw dla firm
Version: alpha 0.1
License: GNU General Public License v2 or later
Tags: clean, bootstrap
*/

Nie wszystki wymienione wyżej elementy są konieczne aby wordpress zatwierdził nasz motyw ale warto je mieć. Po uzupełnieniu i podejrzeniu szczegółów motywu w panelu wordpressa całość będzie wyglądała mniej więcej tak jak na zdjęciu:

Omówienie elementów

  • Theme URI - tutaj wpisujemy adres www naszego motywu, w przykładzie jest to supermotyw.com
  • Author - podajemy autora motywu, w tym przypadku siebie
  • Author URI - adres www autora motywu
  • Description - opis naszego motywu, czego dotyczy etc.
  • Version - podajemy wersję naszego motywu. Obecnie nasz motyw jest na począkowym etapie tworzenia, więc możemy dać mu nazwę alpha 0.1
  • License - licencja na jakies udostępniamy nasz motyw osobm trzecim
  • Tags - tagi, po których będzie można znaleźć nasz motyw w repozytorium wordpressa

Na razie brakuje nam jeszcze miniaturki naszego motywu, zrobimy to później.

Tworzymy nagłówek strony

  1. W katalogu głównym naszego motywu tworzymy plik header.php
  2. Umieszczamy w nim kod nagłówka naszej strony, podobnie jak robi się to przy tworzeniu normalnej strony HTML, jednak nie umieszczamy tutaj żadnych tagów ze stylami lub skryptami, później będzie to wyjaśnione:

    HTML

    <!DOCTYPE html>
  <html lang="pl">
    <head>
      <meta charset="utf-8">
      <title>Super Motyw dla Wordpressa</title>
    </head>
  <body>

Domyślnie nie domykamy elementów (tagów) html i body. Zrobimy to w kolejnym pliku, który zawiera stopkę.

Tworzymy stopkę strony

  1. Tworzymy plik footer.php
  2. Wstawiamy do niego elementy stopki <footer></footer> oraz domykamy elementy <html> i <body> z naszego pliku nagłówka header.php:

    HTML

        <footer>
      <p>Treść naszej stopki</p>
    </footer>
  </body>
</html>

Tworzymy stronę główną

Teraz w naszym folderze z motywem stworzymy kolejny plik o nazwie "index.php". Jak pamiętasz z kursu PHP, jest to plik zawierający kod HTML i PHP, dlatego ma rozszerzenie PHP, aby serwer wiedział jak go zinterpretować. Plik pełni podobną rolę jak index.html - jest plikiem domyślnym, który POPRAWIC jest w pierwszej kolejności wyszukiwany i wczytywany, gdy użytkownik wejdzie na stronę. Stronę index.php wypełnimy przykładową treścią. Musimy także sprawić aby Wordpress doczytał do naszej strony pliki nagłówka header.php i stopki footer.php.

  1. Tworzymy plik index.php
  2. Wypełniamy go przykładową treścią, np:

    HTML

    <h1>To jest moja pierwsza strona w wordpressie</h1>
<p>Wordpress wydaje się trudny, ale tak naprawdę jest prosty</p>

Jeśli teraz uruchomimy nasz motyw i zbadamy kod to zobaczymy, że brakuje naszego nagłówka i stopki:

Aby je dodać musimy w naszym pliku index.php dodać kod, który wywoła funkcję która dołączy treść z pliku header.php do treści naszej strony. To samo zrobimy z plikiem naszej stopki footer.php

  1. W pliku index.php na samej górzej wywołujemy funkcję wczytującą nagłówek: get_header(). Jest to predefiniowana funkcja Wordprewssa, więc Wordpress wie, że natrafiając na nią w kodzie ma wczytać w danym miejscu kod z pliku header.php. Pamiętaj o umieszczeniu kodu PHP w znacznikach PHP <?php ?>
  2. Na dole pliku index.php wywołujemy funkcję wczytującą stopkę: get_footer(). Całość powinna wyglądać tak:

    HTML

    <?php get_header(); ?>
<h1>To jest moja pierwsza strona w wordpressie</h1>
<p>Wordpress wydaje się trudny, ale tak naprawdę jest prosty</p>
<?php get_footer(); ?>

Tworząc strukturę folderów w swoim motywie nie musisz ściśle trzymać się sposóbu jaki został tu przedstawiony, jednak warto się go trzymać, ponieważ pozwala na uporządkowanie plików, utrzymanie czystości, pozwala innym developerom jak i również Tobie na łatwiejszą pracę z Twoim motywem.

W statycznym motywie wszystkie style możemy umieścić w pliku style.css, jednak my zrobimy to inaczej, lepiej. Chcemy aby nasz motyw był skalowalny, z dobrą organizacją i strukturą plików. Teraz stworzymy pliki .css i .js zawierającymi nasze style i skrypty w języku JavaScript.

  1. W głównym katalogu naszego motywu tworzymy foldery css i js, możesz również nazwać te foldery inaczej.
  2. W folderze css (lub z Twoją nazwą), gdzie będą znajdowały się style css do naszego motywu tworzymy plik .css o nazwie motywu, np. supermotyw.css
  3. W folderze js tworzymy plik JavaScript o nazwie naszego motywu, np. supermotyw.js
  4. W pliku supermotyw.css dodajmy jakieś podstawowe style

Teraz musimy dać Wordpressowi informacje, gdzie znajdują się nasze pliki ze stylami i skryptami, by mógł je dołączyć do naszego motywu. Robimy to za pomocą ważnego pliku functions.php, w którym zawrzeć możemy różnego rodzaju instrukcje dla Wordpressa dotyczące naszego motywu. Plik ten musimy utworzyć w głównym katalogu naszego motywu i silnik Wordpressa przy budowaniu strony wczytuje ten plik jako jeden z pierwszych POPRAIW JEŚLI SIĘ MYLĘ i sprawdza czy znajdują się w nim funkcję, które muszą zostać wykonane zanim nasza strona zacznie być tworzona. Plik ten pozwala na manipulowanie, edycję kodem źródłowym wordpressa zanim zacznie on wczytywać inne pliki.

  1. Stwórz w katalogu głównym motywu plik functions.php
  2. Aby dołączyć style z pliku supermotyw.css w nagłówku naszego motywu, w kodzie PHP, pomiędzy tagami <?php i ?> stwórz funkcję 
    function supermotyw_script_enqueue() {
}
    nazwa naszej funkcji ma przedrostek "supermotyw", od nazwy naszego motywu. Staraj się nadawać funkcją unikalną nazwę, ponieważ jeśli będziesz korzystał z pluginu, gdzie twórca użył takiej samej nazwy zmiennej spowoduje, że będziemy mieć różne funkcje o takiej samej nazwie zmiennej co może spowodować nieoczekiwane błędy. Enqueue oznacza dodawać do kolejki, kolejkować.

  4. Wszystkie zaczepy, które znajdują się w funkcji posiadają akronim wp_

Teraz użyjemy haku (ang. hook), funkcjonalności, którą daje nam Wordpress aby załadować style do naszej strony

Haki, czyli akcje i filtry, to rozwiązanie, które pozwala nawet początkującemu programiście bezpiecznie modyfikować niemal każdą funkcjonalność Wordpressa bez dotykania jego kodu źródłowego. Dzięki hakom, aby pisać dodatki, pluginy, motywy do Wordpressa nie trzeba się uczyć skomplikowanego API czy być doświadczonym programistą PHP. Daje to znacznie niższy poziom wejścia dla nowych twórców, co jest właśnie siłą Wordpressa - ogromna społeczność oraz prostota systemu. Istnieją systemy, które są lepsze, bardziej zaawansowane od Wordpressa, ale ich trudność, wyższy poziom wejścia sprawia, że są mało popularne, mają znacznie mniejszą społeczność a tym samym dodatków, wsparcia, rozwiązań.

Każdorazowa odsłona strony opartej na Wordpressie wywołuje następujące akcje: Przy każdym uruchomieniu strony uruchamiane są pliki źródłowe Wordpressa, następnie wszystkie wtyczki, motyw,

  1. Uruchamiane są pliki źródłowe Wordpressa
  2. Uchamiane są wszystkie wtyczki
  3. Uruchamiany jest aktualny motyw
  4. Uruchamiany jest szablon aktalnej strony

Kolejno wykonywane są setki funkcji, które sprawiają, że kiedy przychodzi kolej na szablon konkretnej strony, mamy już do dyspozycji wszystko co potrzeba. Mamy wszystkie ustawienia, wczytane z bazy danych artykuły, komentarze, dane o obrazkach, znamy wszystkie parametry podane w URL, wiemy, czy użytkownik jest zalogowany, a jeśli tak to kim jest. Mamy też do dyspozycji dodatkowe elementy, których dostarczają nam wtyczki i używany motyw. https://wpinternals.pl/programowanie/haki-wordpressa-cz-1-wprowadzenie-do-akcji-i-filtrow/

Aby załadować zewnętrzny arkusz stylów korzystamy z funkcji kolejkującej arkusze stylów wp_enqueue_style()

  1. W środku naszej funkcji supermotyw_script_enqueue wywołaj funkcję wp_enqueue_style(). Jeśli w edytorze masz zainstalowaną wtyczkę ze snippetami do Wordpressa po prostu wpisz wp_enqueue_style i naciśnij TAB. Funkcja automatycznie zostanie wyświetlona z parametrami. Całość powinna wyglądać tak: 
    function supermotyw_script_enqueue() {
  wp_enqueue_style( $handle, $src, $deps, $ver, $media );
}
  2. $handle - (wymagane) nazwa naszego stylu, dzięki której będziemy mogli później się do niego odwoływać (stargetować go). Wszystkie parametry o wartości typu string umieszczamy w apostrofach. Wpiszmy tutaj na przykład customstyle. Jako, że wczytujemy style, Wordpress automatycznie doda to tej nazwy przyrostek -css. Calość będzie wyglądała tak: 
    <link rel="stylesheet" id="customstyle-css" href="https://tutorial.dev/wp-content/themes/supermotyw/css/awesome.css?ver=1.0.0" type="text/css" media="all">
  3. $src - (nie wymagane, jeśli nie mamy stylów we własnym pliku) ścieżka do pliku, który chcemy załadować. Wordpress wymaga ścieżki absolutnej, nie możemy więc wpisać ścieżki relatywnej css/supermotyw.css. Na szczęscie Wordpress daje nam predefiniowaną funkcję get_template_directory_uri(), która pobiera pełną ścieżkę gdzie znajduje się nasz motyw. Zatem jeśli nasza strona ma adres www.nasza-strona.pl, adresem naszego motywu, który będzie pobierany przez funkcję get_template_directory_uri() będzie www.nasza-strona.pl/wp-content/themes/supermotyw. Teraz wystarczy dołączyć do tego adres relatywny, gdzie znajduje się nasz plik css, czyli /css/supermotyw.css. Z lekcji PHP pamiętamy, że konkatenację, czyli wykonujemy przy pomocy kropki:
    get_template_directory_uri() . '/css/supermotyw.css' 
    <link rel="stylesheet"  id="customstyle-css" href="https://tutorial.dev/wp-content/themes/supermotyw/css/awesome.css?ver=1.0.0" type="text/css" media="all">
  4. $deps - inne arkusze od których zależny jest nasz arkusz. W przypadku JavaSCRIPT będzie to np jQuery, które będzie musiałoby być wczytane wcześniej
  5. $ver - (nie wymagane) wersja naszych stylów. Przeglądarki cacheują pliki css i js, jeśli po modyfikacji ich zawartości nie zmienimy ich nazwy, może się zdarzyć, że wyświetlona zostanie strona ze skryptami i stylami sprzed modyfikacji, gdyż zostaną one załadowane z pamięci podręcznej przeglądarki. Ponadto jest to dobra informacja dla nas lub innych pracowników, pracujących nad naszym motywem. Wpiszmy 1.0.0 jako wersję, wordpress automatycznie doda znak zapytania i przedrostek ver. 
    <link rel="stylesheet"  id="customstyle-css" href="https://tutorial.dev/wp-content/themes/supermotyw/css/awesome.css?ver=1.0.0" type="text/css" media="all">
  6. $media - (Optional) The media for which this stylesheet has been defined. Accepts media types like 'all', 'print' and 'screen', or media queries like '(orientation: portrait)' and '(max-width: 640px)'. Default value: 'all'

Całość powinna wyglądać tak 

<?php

function awesome_script_enqueue() {
    wp_enqueue_style('customstyle', get_template_directory_uri() . '/css/awesome.css', array(), '1.0.0', 'all' );
}

Teraz musimy dodać hak wywołujący naszą funkcję supermotyw_script_enqueue. Wykonany to za pomocą haka add_action, który daje nam możliwośc połączenia procesów wykonawczych Wordpressa z funkcjami stworzonymi przez użytkownika.

Defer, niestety Wordpress nie pozwala nam by w funkcji wp_enqueue_script dodać atrybut "defer", dlatego musimy dodać pod nim kolejną funkcję: wp_script_add_data, który przyjmuje 3 parametry. W pierwszym podajemy nazwę celu do ktorego chcemy go zastosować, w naszym przypadku będzie to customjs, podajemy wartość 2 parametru jako 'async', 3 parametr jako true, co oznacza, że nasz atrybut 'async' bedzie dodany.
